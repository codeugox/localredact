Privacy Policy

How Local Redact handles your data.

What we collect

Local Redact has no server, no database, and no analytics. Your documents are processed entirely in your browser using your device's CPU and memory. No data is collected, transmitted, stored, or logged.

Network requests

The application enforces a Content Security Policy with connect-src: 'none', making it technically impossible for the application code to send data to any server after the page loads. You can verify this in your browser's Network tab.

Browser storage

Local Redact does not use cookies, localStorage, sessionStorage, IndexedDB, or any other browser storage mechanism. When you close the tab or click "Start over," all document data is cleared from memory.

Third parties

No analytics, tracking pixels, advertising scripts, or third-party code of any kind is included. No external fonts are loaded. The application uses only system fonts.

Your documents

Your PDF files are processed locally using your device's CPU and memory. The original file is never modified. The redacted output is generated entirely in your browser and downloaded directly to your device.

Open source

This application is open source under the MIT License. You can audit the complete source code to verify these claims.

Disclaimer

Local Redact is provided "as is" without warranty of any kind. While we employ industry-standard rasterization techniques to make redactions irreversible, we make no guarantee that redacted information cannot be recovered through means outside the scope of this application. Always verify the output before sharing sensitive documents.
